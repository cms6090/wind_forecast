# Wind Domain Features — 풍력 도메인 피처 가이드

모든 피처는 "왜 발전량과 관련 있는가"를 물리적으로 설명할 수 있어야 한다. 아래는 풍력발전 예측 문헌에서 표준으로 쓰이는 근거들이다.

## 1. 발전량의 물리학 (피처 우선순위의 근거)

풍력 발전량 공식: **P = ½ · ρ · A · Cp · v³**
(ρ=공기밀도, A=로터 면적, Cp=출력계수, v=풍속)

- **풍속이 3제곱으로 들어간다** → 풍속이 압도적 1순위 피처이고, 풍속 오차가 발전량 오차를 3배로 증폭시킨다. `ws`, `ws²`, `ws³`를 모두 피처로 제공
- 파워커브는 비선형 S자: cut-in(~3 m/s) 이하 발전 0, rated(~11~13 m/s) 이상 정격 출력 고정, cut-out(~25 m/s) 이상 정지. → 구간 더미 변수(저풍속/상승구간/정격/컷아웃)가 유효
- 트리 모델은 v³를 스스로 못 만들지는 않지만 명시적으로 주면 분할 효율이 좋아진다 (표준 관행)

## 2. u/v 성분 → 풍속·풍향 변환 (필수 전처리)

GFS/LDAPS는 바람을 u(동서)/v(남북) 성분으로 준다.

```python
ws = np.sqrt(u**2 + v**2)                    # 풍속
wd = (270 - np.degrees(np.arctan2(v, u))) % 360  # 기상학 풍향 (바람이 불어오는 방향)
```

- **풍향은 원형 변수** → 그대로 쓰면 359°와 1°가 멀어 보임. 반드시 `sin(wd)`, `cos(wd)` 인코딩
- 풍향이 중요한 이유: 산악 지형(태백 가덕산)에서는 풍향에 따라 지형 가속/차폐(wake, 지형 왜곡)가 달라져 같은 풍속이라도 발전량이 다르다 → 풍향×풍속 상호작용 피처 고려

## 3. 높이별 풍속과 윈드시어 (hub height 보정)

터빈 허브 높이는 지상 10 m가 아니다 (info.xlsx의 Hub Height 확인, 보통 80~100 m).

- GFS는 10 m / 80 m / 100 m 풍속을 모두 제공 → **허브 높이에 가까운 80/100 m가 10 m보다 대표성이 높다**
- 윈드시어 멱법칙: v(z) = v(ref) · (z/z_ref)^α. 높이 간 풍속비로 시어 지수 α를 피처화 가능 — 대기 안정도의 대리변수(α가 크면 안정, 야간 역전층)
- LDAPS는 10 m와 50 m max/min 제공 → 50 m max-min 차이는 난류 강도의 대리변수

## 4. 공기밀도 보정

P ∝ ρ이고, ρ = p / (R·T). 겨울(저온·고압)에는 여름보다 밀도가 ~10% 이상 높아 같은 풍속에서 발전량이 더 크다.

```python
rho = sp / (287.05 * t_kelvin)   # 지표기압, 2m 기온으로 근사
ws_density_corrected = ws * (rho / 1.225) ** (1/3)  # IEC 61400-12 방식
```

## 5. 기타 유효 피처 (문헌 근거)

- **돌풍(gust)과 gust/ws 비율**: 난류 강도 대리변수, 출력 변동성과 연관
- **850 hPa 바람**: 산 정상(가덕산 ~1,000 m)은 자유대기 영향권. 850 hPa(~1,500 m) 풍속이 지상 예보보다 산악 풍황을 잘 반영하는 경우가 있음
- **기온, 안정도**: 2 m 기온과 850 hPa 기온 차 → 대기 안정도 지표. 불안정하면 상층 운동량이 지상으로 혼합되어 풍속 증가
- **시간 피처**: hour/month의 sin·cos (일주기·계절 주기). 산악풍은 주야 순환이 뚜렷
- **격자 집계**: 여러 grid의 평균뿐 아니라 std(예보 불확실성 대리), 발전단지 최근접 격자 가중 평균. 격자별 개별 피처 vs 집계 피처는 실험으로 비교
- **예보 시간별 변화율**: 같은 발표분 내 ws의 diff — 전선 통과 등 급변 상황 포착
- **다중 NWP 결합**: LDAPS(고해상도 1.5 km, 지형 반영 우수)와 GFS(전지구, 안정적) 두 모델의 풍속 차이 자체가 예보 불확실성 피처. 앙상블 NWP 결합은 풍력 예측 대회(GEFCom 등) 상위권의 공통 전략

## 6. SCADA 데이터 활용법 (train 전용 주의)

SCADA는 test에 없다 → 직접 피처 금지(leakage-guard 참조). 대신:

1. **파워커브 학습**: SCADA (ws, power) 산점도로 단지 고유 파워커브를 추정 → 예보 풍속을 이 커브에 통과시킨 값을 피처로 사용 (커브는 train으로만 적합)
2. **예보 검증**: 예보 풍속 vs SCADA 실측 풍속 비교로 NWP 편향(bias) 파악 → 편향 보정 근거
3. **터빈 가동률 파악**: 특정 터빈 장기 정지 구간 → 라벨 이상 구간 탐지, 학습 제외 판단 근거

## 7. 그룹 매핑

info.xlsx로 각 KPX 그룹에 어떤 터빈(VESTAS/UNISON, 좌표, 허브높이)이 속하는지 반드시 먼저 확인한다. 그룹별로 최근접 격자·허브높이·파워커브가 다르므로 **그룹별 모델 또는 그룹 피처**가 정당화된다.

## 인용할 만한 근거 (보고서용)

- 파워커브·풍속 3제곱: 풍력공학 표준 교과서(Manwell et al., *Wind Energy Explained*)
- 밀도 보정: IEC 61400-12-1 표준
- NWP 기반 단기 풍력 예측에서 GBDT 계열 우수: GEFCom2014 wind track 상위 솔루션 (Hong et al., 2016, IJF)
- 풍향 sin/cos 인코딩, 시어 지수: 풍력 예측 문헌 전반의 표준 관행
