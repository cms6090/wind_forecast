# Feature Selection — 피처 선택 프로토콜

피처 선택의 유일한 기준은 **동일 CV 구조에서의 검증 점수(src/metric.py)**다. importance 수치는 참고 자료일 뿐 채택 근거가 아니다.

## 1. 진행 순서

1. **베이스 피처셋 확정**: 도메인 근거가 가장 강한 최소 집합 (허브높이 풍속, ws³, 풍향 sin/cos, 시간 sin/cos, 기온·기압) → 이것이 비교 기준
2. **피처군 단위 ablation**: 피처를 하나씩이 아니라 의미 있는 군 단위로 추가/제거 (예: "850hPa 상층군", "격자 std군", "밀도보정군") → fold별 Score 변화 기록
3. 개선 판정: **모든(또는 대다수) fold에서 일관되게 개선 + 개선 폭 > fold 간 표준편차**일 때만 채택
4. 채택/기각 결과를 실험 표에 누적 (HANDOFF.md 실험 기록과 동기화)

## 2. Importance 해석 주의사항

- LightGBM gain importance는 카디널리티 높은 연속 변수에 편향 → 순위 참고용
- **permutation importance를 검증셋에서** 계산하는 것이 더 신뢰 가능 (학습셋 permutation은 과적합된 중요도)
- 상관 높은 피처 쌍(예: 80m ws와 100m ws, 상관 0.99대)은 importance가 서로 나눠져 둘 다 낮아 보일 수 있음 → 쌍 단위로 제거 실험
- importance 0에 가까운 피처라도 제거 전 ablation으로 확인 (트리 모델은 무해한 경우도 많음 → 제거의 이득은 주로 과적합 감소·해석 단순화)

## 3. 다중공선성 정리

- 트리 모델은 다중공선성에 예측 성능상 강건하므로 공격적으로 지울 필요는 없다
- 단, |상관| > 0.98 쌍은 하나만 남기는 것을 기본값으로 (permutation 해석 왜곡 + 노이즈 분할 방지)
- 선형 모델을 베이스라인·앙상블에 쓸 경우에만 VIF 수준의 정리를 별도 수행

## 4. 과적합 신호 감지

- 피처 수가 늘수록 train-valid 점수 격차 추이를 기록한다 — 격차가 벌어지는 피처군은 기각
- 시간 관련 피처(연도, 절대 날짜 등 test 분포에 없는 값)는 금지 — 2025년은 학습에 없는 연도이므로 외삽 실패
- 특정 fold에서만 크게 좋아지는 피처는 우연으로 간주하고 기각

## 5. 그룹별 차등 선택

- 그룹별 모델을 쓰는 경우 피처셋도 그룹별로 다를 수 있다 (최근접 격자, 터빈 제작사 차이)
- 단, 관리 복잡도가 커지므로 그룹별 차등은 공통 피처셋 대비 명확한 개선이 있을 때만

## 6. 산출물

- `04_model_selection.ipynb` 이전에 확정 피처 목록을 `data/processed/train_features_vN.parquet`으로 저장
