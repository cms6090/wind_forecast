# EDA Checklist — 대회 데이터 탐색 순서

EDA는 "그림 그리기"가 아니라 **전처리·피처·검증 설계의 근거 수집**이다. 각 항목마다 "그래서 무엇을 결정했는가(So-what)"를 기록한다.

## 1. 타깃 (train_labels.csv) — 최우선

- [ ] 기간 확인: group_1/2는 2022~2024, **group_3은 2023~2024만** (2022 빈칸) → 학습 데이터 양 차이 확인
- [ ] 분포: 히스토그램. 0 근처 집중(무풍) + 정격 근처 봉우리의 쌍봉 예상. 설비용량 초과값 존재 여부
- [ ] **평가 대상 비율**: 실제 발전량 ≥ 설비용량 10%인 시간대가 전체의 몇 %인가 → 저발전 시간은 평가 제외이므로, 모델이 어느 구간에 집중해야 하는지 결정
- [ ] 시계열 플롯: 월별·계절별 패턴, 장기 결측/0 고정 구간(터빈 점검·고장 의심)
- [ ] 그룹 간 상관: 같은 단지이므로 높은 상관 예상 → 그룹 간 정보 공유 전략 근거
- [ ] 일주기: 시간대별 평균 발전량 (산악풍 주야 순환)
- [ ] 연도별 분포 변화: 2022 vs 2023 vs 2024 (터빈 증설/성능 저하 여부) → 오래된 데이터 가중치 결정

## 2. 기상 예보 (gfs/ldaps)

- [ ] 격자 배치 지도: 위경도 산점도 + info.xlsx 터빈 좌표 오버레이 → 최근접 격자 식별
- [ ] `forecast_kst_dtm` 커버리지: 빠진 발표분(예보 누락일) 존재 여부와 처리 방침
- [ ] `data_available_kst_dtm` 규칙 검증: 항상 전날 13:00인지 실데이터로 확인 (leakage 판단의 전제)
- [ ] 결측: 컬럼별 결측률, 결측이 특정 기간/격자에 몰리는지
- [ ] 풍속 계산 후 분포: 10 m vs 80/100 m, LDAPS vs GFS 비교 (동일 시각 산점도) — 두 NWP 간 괴리 = 불확실성
- [ ] 물리 범위 점검: 기온(K인지 °C인지), 기압 단위, 음수 풍속 불가 등

## 3. 타깃 × 기상 관계 (피처 설계의 핵심 근거)

- [ ] **예보 풍속 vs 발전량 산점도** (그룹별, 격자별): 파워커브 형태 확인. cut-in/rated 위치 추정
- [ ] 풍향별 발전량: 풍향 16방위 구간별 평균 발전량 → 지형 효과 확인
- [ ] 높이별 풍속의 타깃 상관 비교: 10 m vs 80 m vs 100 m vs 850 hPa 중 어느 것이 발전량과 상관이 높은가 → 주력 피처 결정
- [ ] 계절×풍속 상호작용: 같은 풍속에서 겨울/여름 발전량 차이 (공기밀도 효과 검증)

## 4. SCADA (train 전용, 분석 목적)

- [ ] SCADA 실측 풍속 vs 예보 풍속: NWP 편향과 RMSE → "예보가 얼마나 틀리는가"가 이 문제 오차의 하한
- [ ] SCADA (풍속, 발전량) 산점도: 실제 파워커브 → 커브 기반 피처의 근거
- [ ] 이상 패턴: 풍속 높은데 발전 0 (curtailment/정지), 음수 power → 라벨 신뢰도 판단
- [ ] SCADA 합계 vs train_labels 대조: 그룹-터빈 매핑 검증 (info.xlsx 기준)

## 5. 시간 무결성

- [ ] 중복 타임스탬프, 시간 간격 균일성(1시간), DST 없음 확인(KST)
- [ ] train/test 경계(2025-01-01) 전후 데이터 연결 상태
