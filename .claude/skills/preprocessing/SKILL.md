# Preprocessing — 기본 전처리 가이드

전처리의 목표는 "원본 → 시간(8760×연수) 단위의 깨끗한 wide 테이블"이며, **train과 test에 동일한 함수를 적용**하는 것이다.

## 1. 로드 규약

- 모든 CSV는 `encoding='utf-8-sig'` (BOM 존재 — 첫 컬럼명 `\ufeff` 오염 방지)
- 시각 컬럼은 로드 직후 `pd.to_datetime` 변환, 이후 모든 병합 키는 datetime 타입으로 통일
- 로드 직후 검증: 기간 min/max가 data_description.md와 일치하는지, 행 수 일치하는지

## 2. 구조 변환 (long → wide)

- GFS(9격자)/LDAPS(16격자)는 `forecast_kst_dtm` × `grid_id` long 구조 → 시각당 1행이 되도록 pivot 또는 격자 집계
- 1차 캐시는 **격자별 컬럼을 보존한 wide** 버전과 **격자 집계(평균·std·최근접 격자)** 버전 둘 다 저장 → 어느 쪽이 나은지는 모델 비교로 결정 (임의로 하나만 남기지 않는다)
- 병합 순서: labels(또는 sample_submission의 시각 골격)를 기준 축으로 두고 기상 데이터를 left join → 행 손실 방지

## 3. 결측치 처리 원칙

- 처리 전 반드시 결측 지도를 확인 (컬럼별·기간별) — EDA 산출물 참조
- 예보 결측(발표분 누락): 같은 발표분 내 시간 보간만 허용. 발표분 전체가 빠진 날은 직전 발표분 ffill 여부를 leakage-guard로 판정 후 결정
- 라벨 결측: 보간하지 않는다. 학습에서 해당 행 제외 (라벨을 만들어내면 오염)
- group_3의 2022년 공백: 결측이 아니라 "미제공" — 그룹별 학습 구간 분리로 처리
- 모든 결측 처리는 처리 전/후 결측 수를 출력해 기록

## 4. 이상치 처리 원칙

- 물리 범위 기반 필터를 우선한다 (통계적 3σ 컷보다 근거가 명확):
  - 발전량 < 0 또는 > 설비용량 → 검토 후 클리핑/제외
  - 풍속 < 0, 상대습도 0~100 벗어남, 기온 단위 오류(K/°C 혼동) 등
- SCADA 기반 의심 구간(풍속 높은데 발전 0 = 정지/제한발전)은 삭제하지 말고 **플래그 컬럼**으로 보존 → 학습 제외 여부는 검증 점수로 결정
- 이상치를 지운 행 수와 근거를 보고서에 반드시 남긴다 (2차 평가 소명용)

## 5. 단위·좌표 검증

- 기온: GFS/LDAPS가 K인지 °C인지 실측 범위로 확인 후 통일
- 기압: Pa/hPa 통일
- info.xlsx 터빈 좌표와 격자 좌표를 같은 지도에 그려 최근접 격자 ID를 상수로 확정 (그룹별)

## 6. 캐시 산출 규약

- 출력: `data/processed/train_base.parquet`, `test_base.parquet` (+ 격자집계 버전)
- 저장 직전 최종 검증 셀: shape, 기간, 결측 총량, dtypes 출력
- train과 test의 **컬럼 목록이 완전히 일치**하는지 assert (라벨 컬럼 제외) — 불일치 시 추론 단계에서 반드시 사고가 난다
- 캐시에 처리 버전 주석(노트북 셀)과 생성일을 기록

## 7. 금지 사항

- 이 단계에서 스케일링·인코딩·target 기반 변환을 하지 않는다 (fold 내부에서 해야 하는 작업 — leakage-guard)
- 원본 CSV를 절대 덮어쓰지 않는다
