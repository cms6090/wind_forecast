# Model Selection — 베이스라인과 모델 비교

## 1. 베이스라인 사다리 (반드시 아래에서 위로)

복잡한 모델의 개선 효과는 단순한 기준 대비로만 증명된다. 순서대로 만들고 각각의 검증 Score를 기록한다.

1. **시간대×월 평균 예측** (기상 미사용): 이 점수가 "기상 정보의 가치"를 재는 바닥
2. **물리 베이스라인**: SCADA로 추정한 파워커브에 예보 허브높이 풍속을 통과 → ML 없이 도메인만으로 얼마나 가는지. 이후 ML이 이걸 못 넘으면 피처/누수 문제를 의심
3. **선형회귀** (ws, ws³, sin/cos 등 소수 피처)
4. **LightGBM 기본값**: 본 게임의 출발점

각 단계 점수 차가 곧 "무엇이 성능을 만드는가"의 분해이며, 2차 발표 자료의 뼈대가 된다.

## 2. 후보 모델 비교 원칙

- 동일 fold, 동일 피처셋, 동일 후처리(클리핑)에서만 비교한다. 조건이 하나라도 다르면 비교 무효
- 비교 대상: LightGBM / XGBoost / CatBoost / (참고용) RandomForest, Ridge
- 근거: 테이블형·NWP 기반 발전량 예측에서 GBDT 계열이 딥러닝 대비 우세하다는 것이 GEFCom 등 대회 실증의 일관된 결과. 딥러닝(LSTM 등)은 GBDT 대비 명확한 개선을 보일 때만 도입 (재현·설명 비용이 큼)
- 이 단계에서는 **기본 하이퍼파라미터 + early stopping만** 사용. 튜닝은 다음 단계에서 유망 모델에만

## 3. 손실 함수 선택

- 기본: **L1 (MAE)** — NMAE와 정합적 (`objective='l1'` / `reg:absoluteerror` / `MAE`)
- 비교 실험: L2, Huber, quantile(중앙값) — MAE의 최적해는 조건부 중앙값이므로 quantile(0.5)와 동치 관계 확인
- FICR 관점 손실 커스텀은 튜닝 단계 이후의 후순위 실험 (timeseries-validation 스킬 3절)

## 4. 모델 구조 결정 (모델 종류만큼 중요)

다음 세 축을 실험으로 결정하고 근거를 기록한다:

1. **그룹별 3모델 vs 통합 1모델(그룹 ID 피처)**: 통합은 데이터 3배(특히 group_3에 유리), 그룹별은 단지 특성 반영. 둘 다 실험
2. **타깃 스케일**: 원 kWh vs 이용률(발전량/설비용량) — 이용률로 학습하면 그룹 간 통합이 자연스러움
3. **학습 기간 가중**: 2022년 데이터 포함 여부/가중치 (EDA의 연도별 분포 변화 결과에 근거)

## 5. 기록 규약

- 모든 비교는 표로: 모델 / 구조 / fold별 Score·1-NMAE·FICR / 학습시간 / 결론
- 상위 1~2개 모델만 05_tuning.ipynb로 넘긴다
- 오류 분석 미리보기: 최고 모델의 잔차를 풍속 구간별·시간대별로 확인 → 튜닝 단계 방향 제시
