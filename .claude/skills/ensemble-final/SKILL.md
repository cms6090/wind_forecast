# Ensemble & Final — 최종 검증, 앙상블, 재학습

## 1. Seed 안정성 확인

- 최종 후보 모델을 seed 3~5개로 학습 → fold Score의 평균±표준편차 보고
- seed 간 편차가 모델 간 차이보다 크면, 모델 선택 결론 자체를 재검토한다

## 2. 오류 분석 (앙상블보다 먼저)

검증 예측의 잔차를 다음 축으로 분해해 systematic bias를 찾는다:

- 풍속 구간별 (특히 rated 근처와 cut-in 근처 — 파워커브 급경사 구간이 오차 최대 지점)
- 시간대별 / 월별 / 풍향별
- 그룹별 (특히 데이터 적은 group_3)
- **FICR 관점**: 오차율 6~8% 경계에 걸린 시간들의 공통점 (어떤 조건에서 아깝게 이탈하는가)

일관된 편향이 보이면 후보 대응: 해당 구간 피처 보강, 구간별 보정항, 저성능 구간 가중 학습. 각각 검증 점수로만 채택.

## 3. 앙상블

- 우선순위: ① seed 평균 (거의 항상 이득, 비용 0) → ② 이종 모델 평균 (LGBM+CatBoost 등) → ③ 가중 평균 (검증에서 가중 탐색)
- 근거: 앙상블 평균은 분산을 줄여 극단 오차를 감소시킴 → NMAE와 FICR(경계 이탈 확률) 양쪽에 유리
- 스태킹은 시계열 OOF 구성이 까다로워 누수 위험 → 단순 평균 대비 명확한 개선이 없으면 도입하지 않는다
- 앙상블 구성도 실험 표에 기록 (구성 / 가중치 / fold Score)

## 4. 전체 데이터 재학습 (10단계)

- 확정된 피처·파라미터·앙상블 구성으로 **train 전체(2022~2024)** 재학습
- early stopping 문제: 검증셋이 사라짐 → CV에서 관측된 최적 iteration의 평균값(또는 1.05배)으로 `n_estimators` 고정하는 것이 표준 관행. 근거를 주석으로 남긴다
- 재학습 모델의 검증 점수는 없으므로, 제출 전 sanity check로 대체: 예측 분포 vs 2024년 실제 분포 비교, 월별 평균 발전량 형태, 0/설비용량 클리핑 비율

## 5. train.ipynb / inference.ipynb 분리 (2차 평가 필수 요건)

- `train.ipynb`: processed 캐시 로드 → 전체 재학습 → 모델 파일 저장 (`models/`)
- `inference.ipynb`: 모델 파일 로드 → test 캐시 예측 → 후처리 → `validate_submission()` → 제출 CSV 저장
- 두 노트북은 **각각 단독으로, 처음부터 끝까지 오류 없이** 실행 가능해야 한다 (라이브러리 import 포함)
- 시드·파라미터·피처 목록은 하드코딩 또는 저장 파일 로드 — 재실행 시 Private Score 오차 범위 내 복원 필수
- 노트북 상단에 개발 환경(OS, Python, 주요 라이브러리 버전) 기재
