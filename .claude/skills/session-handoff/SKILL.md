# Session Handoff — 세션 인수인계 절차

민석님은 학교와 집을 오가며 작업하므로, 어느 기기에서 열어도 즉시 이어갈 수 있어야 한다. 연결 고리는 **git 저장소 + HANDOFF.md** 두 가지다.

## 1. 세션 시작 절차 (새 세션 첫 행동)

1. `HANDOFF.md`를 읽는다
2. `git log --oneline -5`와 `git status`를 확인한다 (읽기 명령은 AI가 실행 가능)
3. 현재 상태를 3줄 이내로 요약해 민석님에게 보고: "지난 세션에서 ~까지 했고, 다음 할 일은 ~입니다"
4. HANDOFF.md의 "다음 할 일" 첫 항목부터 제안한다

## 2. 세션 종료 절차 ("정리하자" / 작업 단위 완료 시)

1. HANDOFF.md를 아래 템플릿으로 갱신한다
2. 커밋을 **제안**한다 (메시지 문구 포함). `git add/commit/push`는 민석님이 직접 실행한다
3. push까지 되었는지 확인 요청 — push 안 하면 다른 기기에서 이어갈 수 없다

## 3. HANDOFF.md 템플릿

```markdown
# HANDOFF — 마지막 갱신: YYYY-MM-DD HH:MM (장소: 학교/집)

## 현재 위치
- 로드맵 단계: (예: 4. 기본 전처리)
- 작업 중 파일: notebooks/01_preprocessing.ipynb (셀 ~12까지 실행 완료)

## 지난 세션에서 한 것
- (완료 항목 3~5줄)

## 다음 할 일 (우선순위순)
1. (구체적 행동 단위로)
2. ...

## 결정 사항 / 근거 (누적)
- (예: 결측 보간은 같은 발표분 내 선형보간만 — leakage-guard 판정 완료)

## 실험 기록
| 날짜 | 실험 | 로컬 Score | 리더보드 | 결론 |
|---|---|---|---|---|

## 미해결 질문
- (다음 세션에서 토론할 것)

## 환경 메모
- venv 패키지 변경: (있으면 requirements.txt 갱신 여부)
```

## 4. 커밋 제안 규칙

- 타이밍: 노트북 하나 완성, 캐시 생성, 보고서 작성, HANDOFF 갱신 시점
- 메시지 형식 제안: `[단계번호] 요약` (예: `[01] 전처리 노트북: GFS/LDAPS 병합 및 parquet 캐시`)
- 커밋 제안 시 반드시 체크: ① HANDOFF.md 갱신됨? ② `data/processed/`·`venv/`가 .gitignore에 있음? ③ 노트북 출력 정리 여부
- 대용량 파일(parquet, csv 원본)은 커밋 금지 — .gitignore 유지. 캐시는 각 기기에서 노트북 재실행으로 재생성한다

## 5. 기기 간 동기화 주의

- 데이터 원본(`data/`)과 캐시(`data/processed/`)는 git에 없으므로 양쪽 기기에 각각 존재해야 한다. 새 기기 셋업 시: clone → venv 생성 → `pip install -r requirements.txt` → 데이터 복사 → `01_preprocessing.ipynb` 실행으로 캐시 재생성
- 이 셋업 절차가 README.md에 있는지 주기적으로 확인한다
