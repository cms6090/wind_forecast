# Timeseries Validation — 검증 설계와 지표 전략

## 1. 검증 분리 원칙

- test는 2025년 전체 1년 → **검증도 연속된 미래 구간**으로 설계해야 리더보드와 상관이 맞는다
- 기본안: 2022~2023 학습 / 2024 검증 (test와 같은 "1년 전체" 구조, 계절 커버)
- 보강안: 확장 윈도우 시계열 CV
  - fold1: ~2023-06 학습 → 2023-07~12 검증
  - fold2: ~2023-12 학습 → 2024-01~06 검증
  - fold3: ~2024-06 학습 → 2024-07~12 검증
- group_3은 2023년부터만 라벨 존재 → fold 설계 시 그룹별 유효 구간 확인
- 랜덤 K-Fold 절대 금지. 갭(gap)은 이 문제에선 예보 기반이라 필수는 아니나, lag 계열 피처를 쓰게 되면 재검토
- 모든 모델·피처 비교는 **동일한 fold 구조**에서 수행 (비교 가능성 확보)

## 2. 평가지표의 구조적 이해

`src/metric.py` (대회 공식, 수정 금지)를 유일한 채점기로 사용한다.

핵심 특성 3가지:

1. **저발전 시간 제외**: actual < 설비용량 10%인 시간은 채점 안 됨 → 무풍 시간의 오차는 점수 무관. 모델·손실을 발전량 큰 시간대에 집중시키는 근거. 단, 어떤 시간이 제외될지는 사전에 모르므로 예측 시점엔 전 구간을 성실히 예측
2. **NMAE는 설비용량 정규화**: 오차의 절대값 기준. MAE 계열 손실(LightGBM `objective='l1'` 등)이 정합적
3. **FICR은 계단 함수**: 시간별 오차율이 6%/8% 경계를 넘는 순간 정산 단가가 4→3→0으로 급락
   - group_1/2 기준 6% = 1,296 kWh, 8% = 1,728 kWh / group_3 기준 6% = 1,260 kWh, 8% = 1,680 kWh
   - 오차 5.9%와 6.1%는 NMAE로는 거의 같지만 FICR로는 큰 차이
   - 정산금이 actual 가중이므로 **발전량 큰 시간의 6% 이탈이 가장 비싸다**

## 3. FICR 대응 전략 (실험 순서)

1. 우선 NMAE(=MAE)를 낮추면 FICR도 대체로 오른다 — 베이스라인 단계에선 이것만
2. 검증셋에서 **오차율 분포 히스토그램**을 그려 6%/8% 경계 근처 질량을 확인
3. 후보 기법 (검증 점수로만 채택 판단):
   - 분위수 회귀(quantile regression)로 예측 분포를 얻고, 계단 보상을 최대화하는 점 추정 선택
   - 시간대·풍속 구간별 편향 보정 (검증에서 systematic bias 확인 시)
   - 앙상블 평균은 극단 오차를 줄여 경계 이탈 확률을 낮춤 — FICR에 유리한 경향
4. 과최적화 경계: FICR 전용 트릭은 fold 간 일관되게 개선될 때만 채택. 단일 fold 개선은 노이즈로 간주

## 4. 후처리 (항상 적용)

- 클리핑: `pred = pred.clip(0, capacity)` (그룹별 21,600 / 21,600 / 21,000)
- 리더보드 제출 전 `validate_submission()` 필수 통과
- (실험) 저풍속 예보 시간대 0 부근 처리: 파워커브상 cut-in 이하 → 0 예측이 물리적으로 타당한지 검증 점수로 확인

## 5. 실험 관리

- 모든 실험은 표로 기록: 실험명 / 변경점 / fold별 Score·1-NMAE·FICR / 결론
- 베이스라인 대비 개선 폭이 fold 표준편차보다 작으면 "효과 없음"으로 판정
- seed 3개 이상으로 안정성 확인 후 최종 채택
- 리더보드 점수와 로컬 검증 점수를 함께 기록해 상관을 추적한다 (일일 5회 제한 → 로컬 검증 신뢰가 생명)
